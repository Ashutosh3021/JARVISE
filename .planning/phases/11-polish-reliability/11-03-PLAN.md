- AlternativeStrategy class:
  - Define common tool alternatives:
    - "editor" -> ["vscode", "code", "notepad++", "notepad"]
    - "browser" -> ["chrome", "firefox", "edge", "browser"]
    - "terminal" -> ["cmd", "powershell", "wt", "terminal"]
    - "open file" -> Try full path, then search PATH, then common locations
  - get_alternatives(tool_name) -> list of alternative tool names
- RetryEngine class:
  - __init__(tool_registry, max_retries=3)
  - execute_with_retry(tool_name, args) -> RetryResult:
    1. Try original tool with given args
    2. If fails, get alternatives
    3. Try each alternative (up to max_retries)
    4. Record each attempt: {tool, args, success, error}
    5. Return RetryResult with success status and all attempts
  - Should integrate with tool_cache to avoid retrying known failures
  - Log all retry attempts for debugging
  </action>

    - Increment use_count
    - Update last_used timestamp
  - get_preferred(trigger: str) -> Preference | None:
    - Find best matching preference
    - Return highest confidence + use_count preference
  - forget(trigger: str):
    - Remove preference
  - list_preferences() -> list[Preference]
  - Import/export to JSON for backup
- Auto-load on initialization
- Auto-save on any change
  </action>
